# English Brain Training App — MVP 개발 명세서

> 개발자 전달용 문서 · v1.0
> 기반: 기획문서 v2 (전문가 검토 반영)

---

## 1. 프로젝트 개요

한국인 영어 초보자를 위한 **이미지 기반 회화 학습 앱**.
핵심 동사를 "머릿속 그림"으로 형상화하여, 한국어를 들으면 번역 단계 없이
바로 영어가 입에서 나오도록 훈련한다.

**핵심 차별점**
- 동사를 단어가 아닌 "방향 이미지"로 학습 (HAVE=📦 / GET=→📥 / TAKE=✊→ / MAKE=✦)
- 0단계에서 한국어에 없는 개념(관사·be·어순) 선행 학습
- 한국인 예측 가능 실수를 "함정 카드"로 사전 차단
- 콘텐츠가 DB에 분리 → 비개발자가 직접 관리

---

## 2. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 앱 | Flutter | iOS·Android 동시 개발 |
| 백엔드/DB | Supabase (PostgreSQL) | API·인증·관리화면 내장 |
| 콘텐츠 관리 | Google Sheets → 동기화 | 비개발자 친화 |
| 음성 출력(TTS) | 원어민 사전 녹음 파일 | audio_url 저장, 품질 우선 |
| 음성 인식(STT) | 기기 내장 (speech_to_text) | 말하기 평가용 |

---

## 3. 데이터베이스

`db/schema.sql` 참조. 핵심 원칙: **콘텐츠 테이블과 학습 테이블 완전 분리**.

주요 테이블
- `verbs` / `verb_types` / `sentences` — 콘텐츠 (시트로 관리)
- `traps` / `stage0_items` — 실수 예방·뼈대
- `users` / `user_progress` / `daily_missions` — 학습 기록
- `content_version` — 변경 이력(되돌리기)

핵심 함수
- `generate_daily_mission(user_id)` — 하루 5문장 자동 생성 (새 3 + 복습 2)
- `record_answer(user_id, sentence_id, correct)` — 결과 기록 + 간격 반복 복습일 계산

## 4. 화면 구성 (MVP)

### 4.1 홈 / 미션 화면
- 오늘의 미션 카드: "5문장 완성하기"
- 진행 점 5개 (완료 시 초록)
- 연속 학습일·정확도 통계
- 추가 확장 학습 4버튼 (시제/긍정부정/질문/장소시간)

### 4.2 0단계 학습 화면 (신규 사용자 필수)
- 4가지 뼈대를 순서대로 (주어+동사 → be vs 일반동사 → 관사 → "있다" 3종)
- 각 뼈대: 개념 그림 + ❌→⭕ 예시 3~4개
- 완료해야 본 학습 잠금 해제

### 4.3 학습 화면 (핵심)
플로우: **듣기 → 이미지 연상 → 말하기 → 피드백**
1. 동사 타입 태그 표시 (예: HAVE · 소유)
2. 형상화 이미지 + 한국어 상황 (음성 자동 재생)
3. 마이크 버튼 → STT로 사용자 발화 인식
4. 정답 비교 → 맞으면 다음 / 틀리면 힌트→모범답안→따라하기
5. **답 보기 버튼은 5초간 비활성화** (능동 인출 강제)
6. 막힐 때: 함정 카드 표시 (한국인 실수 사전 경고)

### 4.4 완료 화면
- 축하 + 통계 (완료 문장·정확도·연속일)
- 확장 학습 옵션 4개

## 5. 핵심 로직

### 5.1 하루 5문장 선정 (서버 함수 호출)
```
GET /rpc/generate_daily_mission?user_id=xxx
→ 새 문장 3개 (난이도 낮은 것 우선) + 복습 2개 (틀린 것 우선)
```

### 5.2 학습 결과 기록
```
POST /rpc/record_answer { user_id, sentence_id, correct }
→ 정답 시 복습 간격 1→2→4→7→15일로 증가 (간격 반복)
→ 5회 연속 정답 시 mastered = true
```

### 5.3 말하기 평가 (관대하게)
- STT 결과와 정답을 비교하되, 대소문자·문장부호·축약형 무시
- 핵심 단어 70% 이상 일치 시 정답 처리 (초보자 좌절 방지)
- 발음 정확도보다 "말을 시도했는가"에 가중

## 6. UX 원칙 (전문가 검토 반영)

1. **능동 인출**: 답 보기 5초 지연 — 뇌가 먼저 인출
2. **정의적 여과 낮추기**: 틀려도 격려 메시지, 점수 깎지 않음
3. **인지 부하 관리**: 하루 새 문장 3개만, 나머지는 변형·반복
4. **L1 간섭 차단**: 한국어를 어순 힌트(chunk)와 함께 제시
5. **자연스러움**: 모든 음성은 원어민 녹음, 축약형 기본

---

## 7. 개발 우선순위

| 순서 | 작업 | 예상 |
|---|---|---|
| 1 | DB 스키마 생성 (Supabase) | 1주 |
| 2 | 콘텐츠 시트 → DB 동기화 | 1주 |
| 3 | 원어민 음성 녹음·연결 (외주) | 2주 |
| 4 | 앱 MVP — 학습 플로우 | 3~4주 |
| 5 | 나선형 복습 알고리즘 연동 | 2주 |
| 6 | 콘텐츠 관리 화면 (준석용) | 2주 |
| 7 | 베타 테스트·조정 | 계속 |

콘텐츠가 DB에 분리돼 있어 **앱 개발과 콘텐츠 작업 병행 가능**.

---

## 8. 콘텐츠 관리 흐름 (비개발자용)

1. `content/콘텐츠_관리시트.xlsx` 또는 구글시트에서 예문 수정
2. 관리자 페이지 "동기화" 클릭 → 검증(빈칸·중복) 후 DB 반영
3. 변경 전 버전 자동 백업 (`content_version`)
4. 앱 재배포 없이 다음 학습부터 적용
